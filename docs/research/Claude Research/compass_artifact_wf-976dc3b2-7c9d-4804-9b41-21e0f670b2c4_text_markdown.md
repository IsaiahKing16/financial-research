# Conformal Prediction Meets Financial Forecasting: A Practitioner's Guide (2024–2026)

Conformal prediction cannot replace Brier Skill Score as a trading signal gate — but deploying both together creates a two-level quality filter far superior to either alone. BSS remains the only metric that jointly measures calibration and resolution; conformal prediction adds something BSS fundamentally lacks: instance-level uncertainty quantification with formal guarantees, enabling a "trade only when confident" abstention mechanism. The field has evolved rapidly since 2021, with adaptive methods now solving the exchangeability problem that previously made conformal prediction unsuitable for financial time series. This report covers the theoretical foundations, practical tools, and a concrete dual-gating architecture for quantitative trading systems.

## Adaptive conformal inference solved the exchangeability barrier

Standard conformal prediction assumes exchangeability — that data order is irrelevant — an assumption financial time series violently break. Gibbs & Candès (2021) resolved this with Adaptive Conformal Inference (ACI), which treats prediction set construction as an online learning problem. The mechanism is elegant: maintain an adaptive miscoverage level α_t updated after each observation via **α_{t+1} = α_t + γ(α − err_t)**, where err_t indicates whether the true value was covered. When coverage fails, the next interval widens; when it succeeds, the interval narrows. The single parameter γ controls the reactivity-stability tradeoff.

ACI's theoretical guarantee is distribution-free and adversarial: over any interval of length T, **average miscoverage converges to the target α at rate O(1/γT)**, regardless of the data-generating process. This was demonstrated on stock market volatility prediction, where standard conformal prediction catastrophically failed during the 2008 crisis while ACI maintained coverage. The follow-up work by Gibbs & Candès (2024, JMLR) introduced DtACI, which eliminates the need to choose γ by running multiple ACI copies with different learning rates and aggregating via exponential reweighting — achieving strongly adaptive regret bounds across all local time intervals simultaneously.

Zaffran et al. (2022, ICML) independently proposed AgACI, which aggregates ACI instances using the Bernstein Online Aggregation algorithm. A key innovation is separate aggregation of lower and upper interval bounds, producing asymmetric intervals suited to skewed financial distributions. AgACI's coverage has been validated empirically on electricity price forecasting, though it notably lacks formal finite-sample coverage bounds — the theoretical analysis acknowledges this gap.

Several extensions have pushed the frontier further. Angelopoulos, Candès & Tibshirani (2023, NeurIPS) reframed online conformal prediction as **PID control** — proportional tracking handles long-run coverage, integral correction fixes systematic bias, and a "scorecaster" derivative component proactively forecasts future conformal scores rather than merely reacting to past errors. This is the first method to attempt proactive adaptation, though the scorecasting component sacrifices clean theoretical guarantees. Yang, Candès & Lei (2024) proposed Bellman Conformal Inference, using dynamic programming to explicitly minimize interval lengths while maintaining coverage. Bhatnagar et al. (2023, ICML) developed SAOCP with near-optimal strongly adaptive regret **O(D√k) simultaneously for all interval lengths k** — theoretically the strongest guarantee available.

For equity returns specifically, practical limitations remain significant. Heavy-tailed distributions cause ACI to produce excessively wide intervals during tail events. Volatility clustering means intervals lag behind abrupt regime shifts because **ACI is inherently reactive, not proactive**. Fantazzini (2024, JRFM) applied ACI variants to 4,000 crypto-assets for VaR estimation, finding FACI and SF-OGD most precise across quantiles, with ACI algorithms outperforming GARCH models that suffered from convergence failures and structural breaks. A South African equity market study (2025) confirmed the fundamental tension: rolling conformal prediction restores near-nominal coverage but at the cost of interval width that undermines directional trading utility.

## Conformal classification produces prediction sets, not just intervals

While most conformal prediction literature focuses on regression intervals, the classification framework — generating prediction sets with coverage guarantees for discrete outcomes — is directly relevant to UP/DOWN/HOLD financial prediction. The algorithm is straightforward: train any classifier, compute nonconformity scores on a calibration set (typically **s(x, y) = 1 − π̂_y(x)**, the complement of the estimated class probability), find the empirical quantile threshold, then include any label whose score falls below this threshold in the prediction set for new instances.

The Adaptive Prediction Sets (APS) method by Romano, Sesia & Candès (2020, NeurIPS) improved naive thresholding by sorting classes by descending probability and accumulating scores until reaching the true class. This produces scores that are approximately uniform when the model is well-calibrated, enabling better adaptivity across difficulty levels. RAPS (Angelopoulos et al., 2021, ICLR) added regularization that penalizes large prediction sets, reducing set sizes by **5–10× compared to APS** on ImageNet while maintaining exact coverage.

For KNN-based systems like FPPE, natural nonconformity measures already exist in the conformal prediction literature — KNN was in fact the original algorithm in Vovk's foundational work. The classic distance-ratio score computes **α_i = (sum of k nearest same-class distances) / (sum of k nearest different-class distances)**. High ratios indicate the example sits far from its own class relative to others. Alternative measures include the KNN fraction (proportion of k neighbors sharing the predicted class) and leave-one-out variants that account for local density.

Prediction set size becomes the key operational signal. A singleton set **{UP}** indicates unambiguous model confidence; a doubleton **{UP, HOLD}** signals moderate uncertainty; the full set **{UP, DOWN, HOLD}** means the model cannot discriminate — abstain. Wang, Sun & Dobriban (2025) formalized this intuition with SOCOP (Singleton-Optimized Conformal Prediction), designing nonconformity scores that explicitly maximize singleton frequency while maintaining coverage, achieving **20%+ improvement** over standard scores.

Mondrian conformal prediction addresses a critical failure mode for imbalanced financial classes. Standard conformal prediction guarantees only marginal coverage, potentially achieving 90% overall by overcovering frequent classes while severely undercovering rare ones. Ding et al. (2023, NeurIPS) demonstrated this on ImageNet: standard CP at 90% nominal gave only **50.8% coverage for "water jug" but 99.2% for "flamingo."** For financial prediction where HOLD events may be rare, Mondrian CP runs separate calibration per class, ensuring each receives independent coverage guarantees.

Venn-ABERS predictors (Vovk & Petej, 2014) offer a complementary approach, producing calibrated probability intervals rather than prediction sets. The method fits two isotonic regressions on calibration data — one per class in the binary case — yielding an interval [p₀, p₁] for the probability of the positive class. The interval width directly indicates uncertainty. For trading, knowing P(UP) ∈ [0.62, 0.67] is more actionable for position sizing than a bare prediction set. The `venn-abers` Python library supports both binary and multiclass extensions via one-vs-all decomposition.

## Coverage guarantees degrade predictably under regime change

The gap between marginal and conditional coverage is conformal prediction's Achilles heel for financial applications. Marginal coverage averages over all market conditions; a risk manager needs guarantees **conditional on the current regime**. The theoretical impossibility results are sobering: Vovk (2012) proved that object-conditional validity (coverage for each specific x) forces infinite prediction sets; Barber, Candès, Ramdas & Tibshirani (2021) formalized that if the class of conditioning events has VC dimension ≥ 2n+2, approximate conditional coverage is impossible beyond trivial solutions.

Barber et al.'s landmark paper "Conformal Prediction Beyond Exchangeability" (Annals of Statistics, 2023) provides the tightest characterization of coverage degradation. The coverage gap is bounded by the **sum of total variation distances between original and swapped score distributions** — crucially depending on score distributions rather than raw data distributions, meaning a good nonconformity score can provide tight coverage even when raw return distributions shift substantially. Their framework supports exponentially decaying weights w_i = b^{n+1−i}, naturally suited for financial time series where recent observations are more relevant.

Empirical evidence on financial regime transitions quantifies the damage. Schmitt (2025) studied conformal VaR calibration on the CRSP U.S. equity portfolio (1990–2024), finding uncalibrated forecasters showed **5.31% exceedance at a 1% VaR target — a 431% overshoot**. Time-weighted conformal calibration reduced this to 1.09%. Coverage failures clustered in high-volatility quintiles, confirming that marginal coverage masks regime-conditional miscalibration. A COVID-19 supply chain study (2025) found coverage drops ranging from 0% to 86.7% across tasks under identical severe distribution shift — with catastrophic failures correlating with concentrated feature dependence (ρ = 0.714).

Conformalized Quantile Regression (Romano, Patterson & Candès, 2019) partially addresses heteroscedasticity by conformally calibrating conditional quantile estimates, producing intervals that adapt to local volatility. However, CQR still provides only marginal coverage — it doesn't guarantee conditional validity during regime changes. Gibbs, Cherian & Candès (2023) proposed a more principled approach: reformulating conditional coverage as coverage over a class of covariate shifts, achieving exact finite-sample guarantees for overlapping groups. For finance, this enables coverage guarantees conditional on market regime indicators. Calleo (2025) proposed integrating CUSUM-based change-point detection with ACI to dynamically reset calibration windows at structural breaks, proving the conditional coverage gap converges to zero after detected change points.

The calibration window size tradeoff is well-characterized. Shorter windows (~60–100 days) react quickly to regime shifts but produce volatile quantile estimates. Longer windows (~250–500 trading days) give stability but lag during transitions. Empirical evidence suggests the optimal window roughly matches the maximum stationary segment length. Adaptive methods like DtACI sidestep this choice entirely by learning the effective window online — a compelling advantage for production systems where the optimal window changes over time.

## Python libraries have matured into production-ready tools

The open-source landscape has consolidated around several well-maintained libraries, each with distinct strengths.

**MAPIE** (v1.3.0, ~1,400 GitHub stars) is the most comprehensive option and the clear default for scikit-learn users. As part of scikit-learn-contrib, it supports **20+ conformal methods** spanning regression (Jackknife+, CV+, CQR), classification (LAC, APS, RAPS, Mondrian CP), time series (EnbPI), and risk control (Learn Then Test). Classification support is fully mature. Backed by Capgemini and ENS Paris-Saclay, MAPIE has a published 2026 roadmap targeting LLM risk control and enhanced adaptive methods. For a KNN-based system like FPPE, MAPIE's classification module with Mondrian support is directly applicable.

**crepes** (v0.9.0, ~540 stars) by Henrik Boström (KTH) offers a unique feature no other library provides: **Conformal Predictive Systems (CPS)**, which output full cumulative distribution functions rather than just intervals or sets. This enables questions like "what is the probability that the return exceeds threshold τ?" rather than just "is the return in this interval?" crepes also includes conformal test martingales for testing the exchangeability assumption — a diagnostic tool with direct value for detecting regime changes. Classification support was added in v0.7.0 with standard and Mondrian conformal classifiers. The `crepes-weighted` extension adds weighted conformal prediction for covariate shift.

**puncc** (v0.8.0, ~352 stars, Deel/IRT Saint-Exupéry) stands out for its time series support — implementing EnbPI, adaptive EnbPI, weighted split CP for non-exchangeable data, and online sequential split CP. It also offers object detection support (unique among CP libraries) and a HuggingFace tutorial for conformalizing pretrained models. Its two-level API (high-level turnkey + low-level Predictor-Calibrator-Splitter) gives production flexibility.

**TorchCP** (v1.0.3, ~454 stars, SUSTech) is the only library designed for deep learning at scale, with native PyTorch support and GPU acceleration delivering **up to 90% inference time reduction**. It implements the broadest range of classification methods (LAC, APS, SAPS, RAPS, ConfTr, class-conditional CP, ordinal classification) plus conformal methods for graph neural networks and LLMs. For teams using PyTorch-based financial models, TorchCP is the clear choice.

**nonconformist** (v2.1.0, ~421 stars) is effectively abandoned — documentation described as "severely deprecated" with no significant updates. It has been superseded by MAPIE for general use and crepes for research applications.

## BSS and conformal prediction measure fundamentally different things

This is the central question: whether conformal prediction's coverage guarantees can replace or augment BSS > 0 as FPPE's signal quality gate. The answer emerges clearly from decomposing what each metric captures.

The Murphy decomposition of Brier Score gives **BS = REL − RES + UNC**, where reliability (REL) measures calibration error, resolution (RES) measures discrimination, and uncertainty (UNC) is the irreducible base-rate difficulty. BSS = (RES − REL)/UNC, so **BSS > 0 requires resolution to exceed reliability error** — the model must sort outcomes better than climatology despite any calibration imperfection. This jointly tests two distinct prediction qualities in a single metric.

Conformal prediction's coverage guarantee — P(Y ∈ C(X)) ≥ 1−α — is **orthogonal to calibration**. Dabah & Tirer (2024, ICML 2025) directly studied this interplay, finding that temperature scaling improves class-conditional coverage but increases prediction set sizes, revealing a non-monotonic tradeoff. A model with terribly miscalibrated probabilities can achieve valid conformal coverage; the sets will simply be large and uninformative. Conversely, a perfectly calibrated model produces nothing resembling conformal coverage guarantees. As Angelopoulos & Bates (2023) emphasize: "conformal prediction does not improve class separability, sharpen regression accuracy, or improve the performance of a poorly trained model."

Coverage addresses neither BSS component directly. It does not measure calibration because coverage is a property of prediction sets, not probability values. It does not measure resolution because a trivial predictor outputting the full label set {UP, DOWN, HOLD} achieves 100% coverage with zero discrimination. Prediction set size serves as an indirect proxy for resolution — smaller sets indicate better discrimination — but it is a coarser measure that reveals nothing about the probability estimates underlying set construction.

**What conformal prediction uniquely provides is instance-level uncertainty quantification** — something BSS fundamentally cannot deliver. BSS is a retrospective model-level summary; it tells you the model is skillful on average but nothing about whether *this specific prediction* is reliable. Conformal prediction fills exactly this gap. The singleton filter ("trade only when C(x) = {UP}") implements selective prediction with formal guarantees on the selected subset. The Selective Conformal Risk Control framework (2024) formalizes this two-stage approach with theoretical guarantees for both risk and coverage control.

The relationship between set size and Brier Score components is indirect but meaningful. An ISIT 2024 paper formally proved that conformal prediction set size is bounded by a function of the base predictor's generalization error — better models produce smaller sets. Since models with high resolution and low reliability error have better generalization, **high singleton rates correlate with favorable BSS decomposition**, though the relationship is not monotonic or exact.

## The optimal architecture is a dual-gating system

The practical recommendation is unambiguous: use BSS as a model-level quality gate and conformal prediction as an instance-level confidence filter.

**Level 1 — Model quality gate (BSS)**: In each walk-forward window, compute BSS on the validation fold. BSS > 0 confirms the model is skillful — resolution exceeds calibration error. If BSS ≤ 0, shut off trading for this window entirely. This prevents trading with a model that performs worse than always predicting the base rate. No conformal method can replicate this model-level diagnostic because a model with BSS < 0 would still achieve valid coverage — it would just produce mostly non-singleton, uninformative prediction sets.

**Level 2 — Instance quality gate (conformal)**: For each prediction where BSS > 0 has been confirmed, run conformal prediction to generate prediction sets at significance level α. If C(x) = {UP} (singleton), execute the trade. If the set contains multiple labels, abstain. This ensures you only act on predictions where the model's uncertainty is quantifiably low. Use ACI or DtACI rather than standard split conformal to handle non-exchangeability. For FPPE's KNN classifier specifically, the distance-ratio nonconformity score is natural and well-studied.

This dual-gating architecture captures the strengths of both metrics while compensating for their individual blind spots. BSS alone catches bad models but still trades on individually uncertain predictions. Conformal prediction alone filters individual uncertainty but cannot detect when the entire model is unskillful — it would merely reduce trade frequency through larger prediction sets without eliminating low-quality signals. Together, they create a **provably sound two-stage filter**: the model must be skillful (BSS), and each specific prediction must be confident (conformal singleton).

The Mondrian variant adds a third dimension: class-conditional coverage ensures the system doesn't systematically mishandle rare market states. If HOLD events are infrequent, standard CP may achieve 90% marginal coverage while dangerously undercovering HOLD predictions. Mondrian CP with MAPIE's implementation provides separate per-class guarantees, preventing this failure mode.

## Conclusion

The conformal prediction literature has undergone a phase transition since 2021. Adaptive methods (ACI, DtACI, SAOCP, PID control) have eliminated the exchangeability barrier that once made these methods inapplicable to financial time series. Classification support has matured through APS, RAPS, and Mondrian approaches, with production-ready implementations in MAPIE and TorchCP. The theoretical coverage degradation under regime change is now well-characterized — bounded by total variation distances between score distributions, with practical methods (regime-weighted conformal, CUSUM-triggered resets) available to manage it.

For FPPE's specific question, the key insight is that **coverage and calibration live on different axes of prediction quality**. BSS captures the probability axis — are the predicted probabilities accurate and discriminating? Conformal prediction captures the uncertainty axis — how confident is the model about each specific prediction? Neither subsumes the other. The dual-gating architecture (BSS model gate + conformal instance gate) is the theoretically grounded and practically actionable synthesis. Implementation is straightforward: MAPIE's `MapieClassifier` with the APS score and Mondrian categories, applied within FPPE's existing walk-forward framework, using recent validation windows as calibration sets and ACI-style adaptive updates to handle non-stationarity.
