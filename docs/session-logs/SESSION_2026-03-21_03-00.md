### SLE-48 — Vectorized Matcher.query() (DONE)

Replaced per-row `train_db.iloc[indices].copy()` + `project_forward()` with
pure-numpy batch operations on pre-cached arrays.

**Root cause of the bottleneck**: every row in `query()` called
`train_db.iloc[indices].copy()` which rebuilds a DataFrame from scratch —
O(n_probe) pandas overhead per query row.

#### Changes

**`pattern_engine/matching.py`**
- `fit()`: pre-caches `_train_tickers_arr`, `_train_sector_arr`,
  `_train_target_arr`, `_train_ret_arr` as 1D numpy arrays
- `_rebuild_caches()`: new helper — extracts cache logic; called from both
  `fit()` and `engine.load()` (backward compat with saved state)
- `_query_vectorized_batch(distances_b, indices_b, ...)`: new method — pure numpy:
  combined filter mask `(B, n_probe)`, cumsum-based `head(top_k)`, vectorized
  prob_up / mean_ret / signal generation
- `query()`: pre-extracts val arrays once per call; calls
  `_query_vectorized_batch()` per batch; regime fallback per-row only for
  rare under-matched rows (lazy coarse labeler, max 1 init per `query()` call)

**`pattern_engine/engine.py`**
- `load()`: added `engine._matcher._rebuild_caches()` after manual reconstruction

#### Bugs fixed during integration

1. **`_train_target_arr` KeyError** — guard added:
   `if horizon in train_db.columns else np.zeros(...)`

2. **`ArrowExtensionArray` 2D indexing** (`pyarrow.lib.ArrowInvalid: only
   handle 1-dimensional arrays`) — pandas 3.x Arrow-backed string columns
   return `ArrowExtensionArray` from `.values`; these reject `arr[2D_index]`.
   Fixed: `np.asarray(train_db["Ticker"], dtype=object)` in `fit()` and `query()`

3. **Missing caches after `load()`** — `engine.load()` manually reconstructs
   Matcher by setting individual attributes; didn't set new cache arrays.
   Fixed: `_rebuild_caches()` extracted and called from `load()`.
