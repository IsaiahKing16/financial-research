# FPPE Infrastructure Layer — Installation & Usage Guide
# Created: 2026-03-19

---

## What This Is

Three infrastructure systems that work together to make your multi-AI workflow
(Claude Code + claude.ai + Gemini + ChatGPT) faster, more reliable, and resistant
to the hallucination and context-loss problems you've already experienced.

### System 1: Slim CLAUDE.md + Task-Scoped Skills
**Problem solved:** Your PROJECT_GUIDE.md is ~900 lines. Claude Code compliance
degrades past ~100 lines in CLAUDE.md. Skills load on demand — zero tokens when
inactive, full protocol when needed.

### System 2: Campaign Files
**Problem solved:** Multi-session work loses context. Campaign files track scope,
decisions, attempts, and blockers across sessions. Prevents scope truncation
(an agent declaring a 6-phase job done after phase 2) and prevents re-trying
things that already failed.

### System 3: Session Compression (Discovery Relay)
**Problem solved:** Switching between AIs requires re-explaining everything.
Session logs compress a 90-minute session into <60 lines that the next AI can
read in 2 minutes. Target: 80% compression ratio.

## Installation

### Step 1: Copy files to your project root

```cmd
cd C:\Users\Isaia\.claude\financial-research

:: Copy CLAUDE.md to project root (Claude Code reads this automatically)
copy [download-path]\CLAUDE.md .

:: Copy skills directory
xcopy [download-path]\.claude\skills .claude\skills\ /E /I

:: Copy campaign files and templates
xcopy [download-path]\docs\campaigns docs\campaigns\ /E /I
```

Do NOT delete PROJECT_GUIDE.md. It remains the full reference document.
CLAUDE.md is the slim intake; PROJECT_GUIDE.md is the complete source of truth.

```

> /skills
```
You should see: run-walkforward, debug-bss, add-ticker, add-feature-set,
run-backtest, phase2-risk-engine, session-handoff

## How to Use

### In Claude Code Sessions

1. Start a session — Claude Code reads CLAUDE.md automatically (~60 lines)
2. When doing specific work, Claude auto-loads the relevant skill based on
   your request (or you invoke manually with `/run-walkforward`, etc.)
3. Check active campaign: Claude reads `docs/campaigns/PHASE2_RISK_ENGINE.md`
4. End session: invoke `/session-handoff` to generate compressed log

### In claude.ai Sessions

1. Provide PROJECT_GUIDE.md (full reference — claude.ai doesn't have CLAUDE.md)
2. Also provide the active campaign file and most recent session log
3. At end of session, ask Claude to generate a session handoff manually

### When Switching to Gemini or ChatGPT

1. Provide PROJECT_GUIDE.md
2. Provide the active campaign file
3. Provide the most recent 1-2 session logs
4. Include the hallucination anchoring statements from session-handoff skill

### Starting a New Campaign

1. Copy `docs/campaigns/CAMPAIGN_TEMPLATE.md` to a new file
2. Fill in scope, success criteria, out-of-scope items
3. Run the decomposition validation before starting work
4. Update CLAUDE.md §Current Phase to point to new campaign

## Design Principles

1. **CLAUDE.md is the intake, not the home.** Keep it under 80 lines. Everything
   else lives in skills, campaigns, or PROJECT_GUIDE.md.

2. **Skills are triggers, not summaries.** The `description:` field in YAML
   frontmatter tells Claude WHEN to fire. Make it specific.

3. **Campaign files are the single source of truth** for multi-session work.
   If it's not in the campaign file, it didn't happen.

4. **Session logs are disposable but campaigns are permanent.** Logs compress
   individual sessions; campaigns track the full arc.

5. **3-strike rule is enforced by structure.** The "What's Been Tried" section
   in campaigns and session logs makes repeated failures visible.

6. **Hallucination prevention is structural, not behavioral.** Provenance
   requirements are baked into the templates, not just stated as rules.

## Maintenance

- **After each session:** Generate session log via `/session-handoff`
- **After each decision:** Update the active campaign's Decisions Log
- **After each failed attempt:** Update the campaign's "What's Been Tried"
- **When starting a new phase:** Create a new campaign from template
- **When a skill's protocol changes:** Update the skill's SKILL.md directly
- **When locked settings change:** Update BOTH CLAUDE.md AND PROJECT_GUIDE.md
